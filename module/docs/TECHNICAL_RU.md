# NeoRender Engine v1.0.0: техническая архитектура

## Граница воздействия

Модуль изменяет только `debug.hwui.renderer`. Значения `skiagl` и `skiavk` выбирают backend Android HWUI при инициализации нового процесса. NeoRender не изменяет Vulkan HAL, SurfaceFlinger backend, EGL-драйвер, GPU governor, частоты CPU/GPU, thermal policy, SELinux и загрузочные разделы.

## Раннее применение

`post-fs-data.sh` использует `resetprop -n`. Профиль применяется до запуска большинства пользовательских процессов. Уже запущенный процесс не переключается на другой backend.

## OEM baseline

Исходное значение свойства сохраняется вместе с build fingerprint. При смене прошивки baseline снимается заново. Пустое OEM-значение хранится специальным маркером и восстанавливается удалением свойства.

## Renderer-транзакции

Персональный запуск:

1. сохраняет глобальное свойство;
2. создаёт приватную транзакцию;
3. применяет временный renderer;
4. выполняет `force-stop` и запускает новый процесс;
5. пытается получить PID и HWUI pipeline;
6. восстанавливает глобальное свойство;
7. удаляет транзакцию.

Незавершённая транзакция восстанавливается при следующем запуске CLI, в boot-скриптах и при удалении модуля.

## Boot guard и карантин

`post-fs-data.sh` создаёт `boot.pending`. После `sys.boot_completed` служба ждёт стабилизацию и ограниченно контролирует PID SystemUI и глобальное renderer-свойство.

При незавершённой non-Stock загрузке либо подтверждённой нестабильности проблемный профиль помещается в карантин, OEM renderer восстанавливается, `PROFILE` меняется на `stock`, а модуль остаётся включённым. Следующая загрузка проверяет Stock. Автоматический marker `disable` для renderer-сбоев не используется; отключение выполняется только явной пользовательской командой.

## Парный benchmark

A/B-тест создаёт новый процесс сначала с `skiagl`, затем с `skiavk`, сохраняет `gfxinfo framestats`, длительность и тепловые показатели, после чего сравнивает p95 и долю janky frames. Рекомендация создаётся только при достаточной и сопоставимой выборке и привязывается к build fingerprint.

`dumpsys gfxinfo` характеризует Android UI/HWUI и не является универсальным измерителем native-кадров Unity, Unreal Engine или собственного Vulkan/OpenGL ES renderer игры.

## Целостность

`RUNTIME_SHA256SUMS` проверяет файлы, необходимые после установки. `SHA256SUMS` покрывает поставляемые файлы релизного ZIP. Отдельный `.zip.sha256` проверяет сам архив.

## Терминальный интерфейс

`neorender` — POSIX/BusyBox ash-совместимое русскоязычное меню над `neorenderctl`. Оно не содержит отдельной логики изменения renderer и вызывает те же проверяемые CLI-команды.
